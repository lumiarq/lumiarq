<a name="v100"></a>
## v1.0.0

### Service Drivers — real implementations replacing stubs

- `SMTPMailer` — nodemailer-based SMTP driver
- `ResendMailer` — Resend.com transactional email driver
- `BullMQQueue` + `BullMQWorker` — Redis-backed job queue via bullmq
- `LocalStorage` — file-system storage driver
- `S3Storage` — AWS S3 storage driver with presigned URL support
- `RedisCacheDriver` — Redis cache driver via ioredis
- `CronScheduler` — cron job scheduler via node-cron

### New Services

- `NotificationService` — multi-channel notification service (composes mailer + queue)
- `DatabaseAudit` — automatic audit logging to `audit_log` table

### DX Improvements

- `parseBody(req, ZodSchema)` / `parseQuery(req, ZodSchema)` — type-safe request parsing, throws 422 on validation failure
- `defineResource(transformer)` — API resource transformer factory with `.make` / `.collection` / `.paginated`
- `defineExceptionHandler(handler)` — custom global exception handler registration
- `HttpException` hierarchy — `NotFoundError`, `UnauthorizedError`, `ForbiddenError`, `ValidationError`, `BadRequestError`, `ConflictError`
- Per-route throttle: `middleware: ['lumiarq.throttle:5,1']` (max 5 requests per 1-minute window)
- Deprecation/Sunset headers auto-injected on deprecated routes

### New CLI Commands

- `lumis db:seed` — run database seeds
- `lumis db:fresh` — schema push + migrate + seed
- `lumis db:reset` — drop all tables + migrate (no seed)
- `lumis db:studio` — open drizzle-kit studio in the browser
- `lumis publish config <name>` — scaffold a typed config file (`mail`, `queue`, `cache`, `storage`, `session`, `security`, `logging`, `auth`)
- `lumis publish config all` — publish all eight config files at once
- `lumis publish config list` — show what's available and what's already published
- `lumis worker:start [--dev]` — start background worker process
- `lumis worker:list` — list running workers and registered scheduled jobs
- `lumis schedule:list` — list all registered cron jobs with schedule and next run time
- `lumis schedule:run <name>` — run a scheduled job immediately (testing/manual trigger)
- `lumis health` — now runs pre-flight checks (entry.ts, providers.ts, config/app.ts, @types/node, worker.ts) before delegating to `lumis doctor`

### Scaffold Updates

- `bootstrap/middleware.ts` generated by `create-lumiarq-app`
- `bootstrap/worker.ts` generated by `create-lumiarq-app`

<a name="v421"></a>
## v4.2.1

- `contextId` renamed from `requestId` in `ExecutionContext`
- `setApplicationContext(ctx)` added to `@lumiarq/runtime`
- `withTestContext(overrides, fn)` returns zero-arg async fn for Vitest `it()`
- `BaseRepository.connection` now defaults to `'default'`
- `defineModule()` now accepts object API: `{ name, alias?, priority?, prefix?, middleware? }`
- New CLI flags: `make:module --full-stack / --api-only / --domain-only`
- `make:action --with-task` generates 4 files (action + task + DTO + test)

<a name="v400"></a>
## v4.0.0

- `EventBus.dispatch()` replaces `emitAsync` (fire-and-forget void)
- Named routes: `route(name, params?, query?)` with `RouteNotFoundError` on miss
- Route model binding: `defineBinding<T>(resolve)` + `ctx.bound<T>(name)`
- `lumis route:cache` and `lumis route:clear` commands
- `lumis tinker` REPL command
- `@lumiarq/database` promoted to Level 1 (depends on `@lumiarq/core`)

<a name="v340"></a>
## v3.4.0

- `defineRoute()` removed from app-facing API; use `Route.get/post/...()` DSL
- File convention: `*.api.ts` for API routes, `*.web.ts` for web routes
- `@lumiarq/contracts` added: `MailerContract`, `QueueContract`, `StorageContract`, `CacheContract`, `EventBusContract`, `LoggerContract`
- `defineQuery(fn)` added to `@lumiarq/core`
- `lumis make:query <Module> <Name>` command

<a name="v320"></a>
## v3.2.0

- Auth starter: 34 auth + 25 user + 11 email verification stubs
- IAM bounded context (`lumis auth:install --iam`)
- `key:generate` generates RS256 4096-bit key pair
- 21 `make:*` scaffold commands
- `@lumiarq/query` framework-agnostic data fetching: `/react`, `/vue`, `/solid`, `/svelte`
